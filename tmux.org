* Terminal type

Force tmux to use true color:
#+begin_src conf :tangle ~/.tmux.conf
  set -g default-terminal "xterm-256color"
  set -ag terminal-overrides ",xterm-256color:RGB"
#+end_src

Ideally, tmux should detect the terminal emulator's color capabilities
automatically. In practice, this does not always work, so we may need to
explicitly tell tmux that the terminal emulator supports true color.

* Prefix key

Use =F12= as the tmux prefix key. If this does not work, check the value of
"TERM". Try setting it to "xterm" to see if the issue goes away.
#+begin_src conf :tangle ~/.tmux.conf
  set -g prefix F12
#+end_src

Pressing =F12= after a prefix key (=F12=) sends the prefix key to the target
window.
#+begin_src conf :tangle ~/.tmux.conf
  bind-key F12 send-prefix
#+end_src

This makes nested tmux sessions easier to use:
- =F12 <key>= sends =<key>= to the outer tmux session.
- =F12 F12 <key>= sends =<key>= to the inner tmux session.

* Nested tmux sessions

- Press Shift+F12 once :: Enter the "nested mode": the outer session's status
  bar is hidden, and all subsequent keys (except for Shift+F12) are passed through
  to the inner session, so it behaves as if it were not nested.

- Press Shift+F12 again :: Exit nested mode and restore the outer session.

#+begin_src conf :tangle ~/.tmux.conf
  bind -T root S-F12 set prefix None \; set key-table off \; set status off \; refresh-client -S
  bind -T off S-F12 set -u prefix \; set -u key-table \; set status on \; refresh-client -S
#+end_src

* Editing style

Use Emacs key bindings in the status line and in copy mode.
#+begin_src conf :tangle ~/.tmux.conf
  set -g status-keys emacs
  set -g mode-keys emacs
#+end_src

Use =C-g= (instead of =ESC=) to quit copy mode and command editing, matching
Emacs.
#+begin_src conf :tangle ~/.tmux.conf
  bind-key -T copy-mode-emacs C-g send-keys -X cancel
  bind-key -T edit-mode-emacs C-g send-keys -X cancel
#+end_src

* Status bar

#+begin_src conf :tangle ~/.tmux.conf
  set -g status on
  set -g status-position bottom
  # Key to toggle status bar on/off
  bind-key b set status

  # colors
  set -g status-style 'bg=#1e1e2e fg=#cdd6f4'

  # Left side
  set -g status-left-length 60
  set -g status-left '#[fg=#1e1e2e,bg=#a6e3a1,bold] #H #[fg=#89b4fa,bg=#1e1e2e] | #[fg=#1e1e2e,bg=#89b4fa,bold]#{?session_grouped, #{session_group}:,} #S #[fg=#89b4fa,bg=#1e1e2e] | '
  set -g status-justify left # the window list

  # Right side of status bar
  set -g status-right-length 0
  set -g status-right ''

  # Window status
  set -g window-status-format ' #I:#W '
  set -g window-status-current-format '#[fg=#1e1e2e,bg=#cba6f7,bold] #I:#W #[fg=#cba6f7,bg=#1e1e2e]'
  set -g window-status-activity-style fg=red

  # Window Behavior
  setw -g automatic-rename on
  setw -g monitor-activity on
  set -g status-interval 1
#+end_src

* Panes

Split panes while keeping the current directory, using more mnemonic keys than
the defaults (=%= and ="=):
#+begin_src conf :tangle ~/.tmux.conf
  bind-key | split-window -h -c "#{pane_current_path}"
  bind-key _ split-window -v -c "#{pane_current_path}"
#+end_src

Cycle panes with =Tab=, which is easier to key-chord than =F12 <arrow>=:
#+begin_src conf :tangle ~/.tmux.conf
  bind-key -r Tab select-pane -t:.+
  bind-key -r BTab select-pane -t:.-
#+end_src

Swap, join, and synchronize panes:
#+begin_src conf :tangle ~/.tmux.conf
  bind-key C-s setw synchronize-panes
  bind-key m command-prompt -p "move (swap) current pane with: " "swap-pane -s %1"
  bind-key j command-prompt "join-pane -h -d -s '%1'"
#+end_src

Cycle through the predefined layouts:
#+begin_src conf :tangle ~/.tmux.conf
  bind-key -r Space next-layout
#+end_src

** Resizing
#+begin_src conf :tangle ~/.tmux.conf
  # Resize panes using Prefix + Shift + Arrow keys
  bind-key -r S-Up resize-pane -U 5
  bind-key -r S-Down resize-pane -D 5
  bind-key -r S-Left resize-pane -L 5
  bind-key -r S-Right resize-pane -R 5
#+end_src

* Windows

Create windows in the current directory, and navigate between them with
repeatable keys:
#+begin_src conf :tangle ~/.tmux.conf
  bind-key c new-window -c "#{pane_current_path}"
  bind-key -r n next-window       # repeatable versions of the defaults
  bind-key -r p previous-window
#+end_src

Re-enable automatic window renaming (handy after tmux turns it off once a window
has been renamed manually):
#+begin_src conf :tangle ~/.tmux.conf
  bind-key < set automatic-rename on
#+end_src

Open a new window running a given command:
#+begin_src conf :tangle ~/.tmux.conf
  bind-key C command-prompt "new-window '%1'"
#+end_src

* Sessions

Create a named session:
#+begin_src conf :tangle ~/.tmux.conf
  bind-key S command-prompt "new-session -s '%1'"
#+end_src

* Copy mode & clipboard

** Page scrolling

=PageUp= enters copy mode and scrolls up, unless an application is using the
alternate screen (in which case the key is passed through). This mirrors the
mouse-wheel behavior.
#+begin_src conf :tangle ~/.tmux.conf
  bind-key -T root PPage if-shell -F "#{alternate_on}" "send-keys PPage" "copy-mode -e; send-keys PPage"
#+end_src

** OS Clipboard

*** App copies

Accept OSC 52 clipboard writes from applications running inside tmux panes
(for example, Emacs kills). Store them in the tmux buffer and forward them to
the outer terminal (wezterm → macOS clipboard).

#+begin_src conf :tangle ~/.tmux.conf
  set -g set-clipboard on
#+end_src

*** Save macOS clipboard

Bind =C-]= to paste the current macOS clipboard contents *and* save them into a
tmux buffer so that the contents are available as a tmux buffer for later reuse.

For ordinary one-off clipboard pasting, use the terminal or OS paste shortcut
instead, such as Cmd-V in wezterm.

This is intentionally not bound to =]=: tmux buffers are mainly for CLI
copy/paste, while the system clipboard may contain large documents or source
code that should not be pasted into a shell accidentally.
#+begin_src conf :tangle ~/.tmux.conf
  if-shell 'command -v pbpaste' {
    bind-key C-] run "tmux set-buffer \"$(pbpaste)\"; tmux paste-buffer"
  }
#+end_src

** Pre-loaded copy buffers

Tweak this section to pre-load as many tmux copy buffers as needed.
#+begin_src conf :tangle ~/.tmux.conf
  # pre-load a frequently used text into the buffer named "hw"
  set-buffer -b hw "hello world"
#+end_src

** Save screen log

Bind =C-h= to save the whole scrollback of the current pane to a file: it enters
copy mode, selects everything, and writes it out.
#+begin_src conf :tangle ~/.tmux.conf
  bind-key C-h command-prompt -p "save log to:" \
           "copy-mode; send-keys 'M->' C-e C-space 'M-<' C-a C-w; save-buffer %%"
#+end_src

The version below is the equivalent for =mode-keys vi=:
#+begin_src conf :tangle no
  bind-key C-h command-prompt -p "save log to:" \
           "copy-mode; send-keys g 'space' G 'enter'; save-buffer %%"
#+end_src

=pipe-pane= can log a pane continuously to a file, but it is not recommended: it
captures raw output (including control sequences) and is easy to leave running
unnoticed.
#+begin_src conf :tangle no
  bind-key h pipe-pane "cat >>$HOME/#S-#I-#P.log" \; \
           rename-window "log" \; \
           display-message "Start logging($HOME/#S-#I-#P.log)"
  bind-key H pipe-pane \; \
           setw automatic-rename on \; \
           display-message "Stopped logging(#S-#I-#P.log)"
#+end_src
