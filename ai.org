** User level configuration file: =~/.claude/settings.json=

- =~/.claude/settings.json= :: User-level configuration file. Applies globally
  to Claude Code sessions for the current operating-system user, unless
  overridden by project-level or local settings.

- =~/.claude/remote-settings.json= :: Remote or account-managed user
  configuration file. Treat this as policy managed by the organization: local
  changes are reverted when a Claude session starts.
  + Note: Claude appears to load permission lists before reverting this file, so
    edits affect the first session after the change.

- =~/project_root/.claude/settings.json= :: Project-level team-shared
  configuration file, meant to be checked in version control.

- =~/project_root/.claude/settings.local.json= :: Project-level personal
  configuration file, meant to be git ignored. Claude Code writes to this file
  every time we grant a permission interactively, so when it's checked in we get
  constant unwanted diffs.

#+begin_src json-ts :tangle "~/.claude/settings.json" :noweb yes :mkdirp yes
  {
    "includeCoAuthoredBy": false,
    <<claude-permissions>>,
    <<sandbox>>
  }
#+end_src

** COMMENT Use AWS Bedrock Models

1. Add the following to =~/.claude/settings.json=:
   #+begin_src json-ts
     {
       "env": {
         "CLAUDE_CODE_USE_BEDROCK": 1,
         "ANTHROPIC_MODEL": "us.anthropic.claude-sonnet-4-20250514-v1:0",
         "AWS_PROFILE": "ai-coder-llm",
         "AWS_REGION": "us-east-1"
       }
     }
   #+end_src

2. Authenticate with AWS:
   #+BEGIN_SRC bash
     aws sso login --profile ai-coder-llm
   #+END_SRC

3. Verify the authentication information
   #+begin_src bash
     aws sts get-caller-identity --profile ai-coder-llm
   #+end_src

4. Run Claude to confirm that everything works.
   #+begin_src bash
     claude
   #+end_src
