* Adjust System Settings

- Mouse:
  + Turn on "Natural scrolling"
  + Enable "Secondary click"
- Keyboard
  + Adjust "Key repeat rate" and "Delay until repeat"
  + Dictation:
    - Languages: select both English and Chinese
    - Shortcut: set to press right command twice
- Mouse/trackpad tracking speed: adjust
- Function Keys: turn on "Use F1, F2, etc. keys as standard function keys"
- Language input methods: add additional input methods, say "Pinyin -
  Simplified"
- Menu Bar
  + "Automatically hide and show the menu bar": set to "Always"
    - To reveal it manually using keyboard, press =Control+F2=, which moves
      keyboard focus to the menu bar.
  + Menu Bar Controls:
    * "Sound": change to "Always Show"
    * "Bluetooth": tick
- Remote Login: toggle on/off as needed

** Weekend battery drain in sleep

NOTE: This is an *unresolved* issue. Documented here for future reference.

*** Symptom

A fully charged MacBook is closed and left in a bag on Friday, then is dead by
Sunday. Seen on the new M5 MacBook Pros.

*** Root cause

While asleep on battery the machine dark-wakes ~220–250 times/hour (~1.9%/hour
drain -> dead in ~2 days). Two distinct sources, found via =pmset -g log=:
1. Power Nap background tasks (~dasd~ jobs: Find My ~searchpartyd~, Jamf Connect
   retries, widgets, Photos, Apple Intelligence). Partly fixable — see below.
2. The Wi-Fi radio waking the CPU for *incoming* network packets (wake reason
   ~wifibt SMC.OutboxNotEmpty centauri-beta~; some tagged ~E_RX_IP_PACKET ARPT~
   / ~E_PFN_NET_FOUND ARPT~). This is ordinary LAN broadcast/multicast chatter.
   NOT fixable via any setting.

The CPU is awake only ~10 min across an entire night (median ~0.3s/wake), so the
drain is the Wi-Fi *radio* powering up ~230×/hour, not computation.

*** What helped

Disabling Power Nap (source #1) is worth keeping — it cut drain from ~4.5 to
~1.9%/hour — but does NOT stop the Wi-Fi wakes:
#+begin_src bash
  sudo pmset -b powernap 0        # battery only (-b); AC profile unchanged
#+end_src

*** What we tried that did NOT work

- ~sudo pmset -b tcpkeepalive 0~ :: verified ~TCPKeepAlive=disabled~ in the
  sleep logs over several nights; the wake storm continued unchanged. It only
  stops *outbound* keepalives, not the radio waking for *inbound* packets.
- ~sudo pmset -b womp 0~ :: no effect (~womp~ is Wake-on-LAN magic-packet only;
  on battery it is already 0 by default on notebooks).
- ~sudo pmset -b darkwakes 0~ :: NOT a valid key on this macOS (Tahoe). ~pmset~
  accepts it silently, but it never appears in ~pmset -g custom~ and has no
  effect.
- Unpairing Bluetooth devices (AirPods, Magic Keyboard) :: no effect — zero
  Bluetooth-tagged wakes, confirming the cause is Wi-Fi, not BT.

*** Diagnostics

#+begin_src bash
  # both Battery + AC profiles (plain -g shows only the currently active source):
  sudo pmset -g custom
  # wake history — bucket by hour, check Charge:N% and "Using AC/BATT":
  pmset -g log | grep -iE "DarkWake|^Wake "
#+end_src

The same Wi-Fi dark-wake sleep drain is widely reported by others on M5 / Apple
Silicon MacBooks.

* Setup GnuPG (GPG)

** macOS: Use the GUI pinentry

1. Install =pinentry-mac=:

   #+begin_src bash
     brew install pinentry-mac
   #+end_src

2. Configure =gpg-agent= to use =pinentry-mac= by adding the following to =~/.gnupg/gpg-agent.conf=:

   #+begin_src snippet :tangle ~/.gnupg/gpg-agent.conf :mkdirp yes
     pinentry-program /opt/homebrew/bin/pinentry-mac
   #+end_src

3. Restart =gpg-agent=:

   #+begin_src bash
     killall gpg-agent
   #+end_src

** macOS: Disable saving passphrases to Keychain

Do not select =Save in Keychain= in the =pinentry-mac= dialog. To disable
Keychain integration by default, run:
#+begin_src sh
  defaults write org.gpgtools.pinentry-mac DisableKeychain -bool YES
  defaults write org.gpgtools.common DisableKeychain -bool YES
#+end_src

** Cache passphrases for longer

Add the following settings to =~/.gnupg/gpg-agent.conf=:
#+begin_src snippet :tangle ~/.gnupg/gpg-agent.conf :mkdirp yes
  default-cache-ttl 28800
  max-cache-ttl 8640000
#+end_src
