This repository contains my configuration files for Emacs, Tmux, window manager,
shell and other tools, organized using /literate configuration/.

* Why literate configuration?

Literate configuration keeps source code and documentation together in a
human-readable document, making the configuration easier to understand and
share, and keeps the document as the single source of truth. A build step called
/tangling/ extracts source blocks from the document and writes them to the
configuration files used by each application.

Writing the configuration in Emacs Org mode also lets us:

- write comprehensive, structured notes alongside the configuration code

- reorganize sections easily by moving, promoting, or demoting headings

- tangle one document into multiple related configuration files

  For example, [[file:shell.org][shell.org]] generates the main Fish configuration as well as
  all the Fish function scripts.

- define flexible tangling rules

  For example, we can configure a source block to be tangled only on my personal
  machine:
  #+begin_example
  :tangle (if (bound-and-true-p my/work-machine-p) "no" "~/.config/opencode/opencode.json")
  #+end_example

* Generating and updating configuration files

If you do not use Emacs, copy and paste the source blocks you need into the
appropriate configuration files manually. Emacs users can use Org Babel to
tangle the source blocks into their designated files, either manually or
automatically as explained below:

** Generate configuration files manually

Download the designated org file, say =shell.org=, open it in Emacs and then run
=M-x org-babel-tangle=. Corresponding configuration file(s) will then be
generated at the specified path(s).

** Update configuration files automatically

To tangle an Org file whenever you save it, add =org-babel-tangle= to its
=after-save-hook=. See [[file:emacs.org::* Automatically tangle configurations][Automatically tangle configurations]] in my [[file:emacs.org][Emacs config]].

* Protecting sensitive data

To avoid accidentally pushing sensitive data to GitHub, install a pre-push hook
to block ourselves from pushing the branch =mine=. This allows us to store
sensitive data in the =mine= branch without the concern of that branch being
accidentally pushed.

For the details, see [[file:git.org::* Block pushing certain branches][Block pushing certain branches]] in [[file:git.org][my git config]].
